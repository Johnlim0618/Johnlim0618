Logging and Monitoring Policy
- Authentication events must be centrally collected
- High-value systems must retain logs for a defined retention period
- PowerShell and administrative activity should be logged where possible
- Alerts with repeated failed authentication attempts must be reviewed
- Security-relevant logs should be protected from unauthorized modification
