Sigma Rule Testing and Conversion

This page includes a simple Sigma-style rule for suspicious PowerShell usage. It demonstrates how portable detection content can be written before being converted into a SIEM-specific query.

What this project shows

  • Sigma rule logic
  • ATT&CK-aligned detection thinking
  • Portable content before vendor-specific conversion

Example artifact

title: Suspicious PowerShell Encoded Command
id: 7f0f4e01-1111-2222-3333-444455556666
status: experimental
logsource:
  product: windows
  service: powershell
detection:
  selection:
    ScriptBlockText|contains:
      - "EncodedCommand"
      - "FromBase64String"
  condition: selection
level: high
